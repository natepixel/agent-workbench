# Contributing

Contributions are welcome. To keep the template coherent:

- **Follow repo conventions** — See `AGENTS.md` for how AIs and humans are expected to work in this repo.
- **Preserve `./scripts/dev.sh`** as the stable public entrypoint; put repo-specific logic in `scripts/dev.repo.sh`.
- **Put temporary work in `working/`** — Drafts, experiments, and handoff docs belong there; promote to `skills/` or top-level docs when stable.
- **Update docs when changing conventions** — If you change how scripts, `working/`, or `skills/` work, update `AGENTS.md`, `README.md`, and any affected READMEs.

Thank you for considering a contribution.
